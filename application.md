**[ Название проекта ]**

#### FocusCore

**[ Проблемное поле ]**

Преподаватели, рассказывающие материал на достаточно большую аудиторию, не могут точно определять уровень внимания студентов на их занятии. Человеческий мозг может эффективно отслеживать лишь около 3-5 движущихся объектов одновременно. Следовательно, в аудиториях с большим количеством слушателей (50–100 человек) преподаватель физически не может провести точную и оперативную оценку вовлеченности каждого студента. Кроме того, оценка все равно остается субъективной, а из за невозможности непрерывного сбора метрик, ее достоверность снижается. Преподаватель, осознавая проблему, вынужден игнорировать низкий уровень вовлеченности студентов из за невозможности опереться на точные данные.
Согласно исследованиям, когнитивное блуждание во время прослушивания лекции критически снижает качество усвоения материала. Уровень поведенческой вовлеченности определяет восприятие и успеваемость студентов, что делает его мониторинг критически важным для эффективности обучения. Из этого можно сделать вывод о необходимости автоматизированного решения, нацеленного на сбор объективных метрик внимания студентов, не требующего затраты лекционного времени. Также оно должно гарантировать безопасность обработки и хранения данных.

К нему можно предъявить следующие требования :
* Подключение к внешней видеокамере и захват видео потока, обработка локально для избежания задержек
* Обнаружение ключевого признака (ключевые точки головы/ глаз/ лица) для извлечение одной простой метрики (угол поворота головы/ открыты ли глаза/ наличие лица в кадре)
* Выполнение классификации на основе данных признаков для отображения текущего статуса коллективной группы, после выявления состояния вовлеченности каждого из студентов
* Сохранение всех измерений в виде отчета со статистикой внимания

Во всех существующих системах для оценки уровня вовлеченности, по нашим данным отсутствуют :
* Анализ длительного поведенческого контекста
* Open-source решение со свободной лицензией
* Генерация персонализированных отчетов для преподавателей

Заявленный программный продукт нацелен решить данные проблемы и обеспечить непрерывный мониторинг, объективные метрики и возможность корректировки методики преподавания.

**[ Функциональные требования ]** 
Программный продукт должен :
* Стабильно и точно определять уровень концентрации аудитории, используя слияние нескольких признаков (положение головы, ключевых точек лица, направление взгляда) по временному ряду, вместо оценки отдельных кадров. Это дает возможность различать концентрацию от скуки, повышая робастность системы. 
* Давать обратную связь в реальном времени, в виде показателя коллективного уровня вовлеченности студентов на дашборде (цветовые индикаторы, отображающие высокий/ средний/ низкий уровень вовлеченности соответственно).
* Давать структурированный отчет по окончании работы, в который будет включен график зависимости вовлеченности студентов от времени лекции. Также данный отчет должен содержать моменты критического падения внимания с приложенными аудио-фрагментами, чтобы преподаватель имел возможность провести рефлексию для последующей смены тактики преподавания.

**[ Похожие продукты ]**
* Системы интерактивного голосования, такие как [iClicker](https://www.iclicker.com), [Mentimeter](https://www.mentimeter.com), [Vevox](https://www.vevox.com) : требуют участия студентов и позволяют оценить вовлеченность аудитории только в момент прохождения опроса, предоставляя лишь фрагменты данных о понимании всей теории лекции.
* [Affectiva](https://www.affectiva.com) (система анализа эмоций) : основывается на одномоментных эмоциях, что не является прямым и достаточно надежным показателем уровня вовлеченности. Эмоции по типу радости или удивления могут быть вызваны сторонними факторами, вместо материала лекции, что ведет к недостоверным метрикам.
* [EduSense](https://www.cmu.edu/news/stories/archives/2019/november/edusense-fitbit-your-teaching-skills.html) : требует множество сенсоров для сбора данных (инфракрасные и видеокамеры, микрофоны), для последующей покадровой/поминутной обработки, вместо анализа длительного поведенческого контекста.

**[ Возможные риски ]**
* Производительность: использование большого количества признаков (поза человека, ключевые точки лица) может привести к падению частоты кадров
* Аппаратные ограничения: локальная обработка может потребовать слишком больших размеров оперативной памяти
* Неэффективность дополнительных параметров: эксперименты могут показать незначительное улучшение, которое не сможет оправдать использование всех запланированных признаков
