**[ Название проекта ]**

#### FocusCore

**[ Проблемное поле ]**

Преподаватели работающие в очном формате не могут точно определять уровень вовлеченности студентов. Лектор может интуитивно понять общее настроение аудитории, но он физически не может одновременно вести занятие, следить за собственными материалами и непрерывно оценивать уровень внимания аудитории. Кроме того, его оценка все равно остается субъективной, а из за невозможности непрерывного сбора метрик, ее достоверность снижается. Преподаватель, осознавая проблему, вынужден игнорировать низкий уровень вовлеченности студентов из за невозможности опереться на точные данные.
Согласно [исследованиям](https://pubmed.ncbi.nlm.nih.gov/17073528/), когнитивное блуждание во время прослушивания лекции критически снижает качество усвоения материала. Уровень поведенческой вовлеченности определяет восприятие и успеваемость студентов, что делает его мониторинг критически важным для эффективности обучения. Из этого можно сделать вывод о необходимости автоматизированного решения, нацеленного на непрерывный сбор объективных метрик внимания студентов, не требующего затраты лекционного времени. Это все дает фундамент для проведения преподавателем последующей рефлексии. 


К нему можно предъявить следующие требования :
* Подключение к внешней видеокамере и захват видео потока, обработка локально для избежания задержек
* Обнаружение ключевого признака (ключевые точки головы/ глаз/ лица) для извлечение одной простой метрики (угол поворота головы/ открыты ли глаза/ наличие лица в кадре)
* Выполнение классификации на основе данных признаков для отображения текущего статуса коллективной группы, после выявления состояния вовлеченности каждого из студентов
* Сохранение всех измерений в виде отчета со статистикой внимания

Во всех существующих системах для оценки уровня вовлеченности, по нашим данным отсутствуют :
* Анализ длительного поведенческого контекста
* Open-source решение со свободной лицензией
* Генерация персонализированных отчетов для преподавателей

Заявленный программный продукт нацелен решить данные проблемы и обеспечить непрерывный мониторинг, объективные метрики и возможность корректировки методики преподавания.

**[ Функциональные требования ]** 

Программный продукт будет предоставлять следующие возможности: 

* Выбрать и подключиться к внешней видеокамере. 
* Включить аудиозапись для сохранения данных в моменты критического падения внимания.
* Управлять записью лекции (начать, приостановить, завершить).
* Предоставить уровень вовлеченности студентов, на основании автоматически извлеченных поведенческих признаках (положение головы, поза тела), по временному ряду, вместо оценки отдельных кадров.
* Предоставить данные в реальном времени, в виде показателя коллективного уровня вовлеченности студентов на дашборде (цветовые индикаторы, отображающие высокий/ средний/ низкий уровень вовлеченности соответственно).
* Сгенерировать и получить структурированный отчет по окончании работы, в который будет включен график зависимости вовлеченности студентов от времени лекции. 
* Предоставить в виде отчета моменты критического падения внимания с приложенными аудио-фрагментами, чтобы преподаватель имел возможность провести рефлексию для последующей смены тактики преподавания.
* Просматривать, сохранять, удалять историю предыдущих сессий
* Самостоятельной установки минимального порога уровня вовлеченности, для отслеживания моментов его критического падения. 


**[ Похожие продукты ]**
* Системы интерактивного голосования, такие как [iClicker](https://www.iclicker.com), [Mentimeter](https://www.mentimeter.com), [Vevox](https://www.vevox.com) : требуют участия студентов и позволяют оценить вовлеченность аудитории только в момент прохождения опроса, предоставляя лишь фрагменты данных о понимании всей теории лекции.
* [Affectiva](https://www.affectiva.com) (система анализа эмоций) : основывается на одномоментных эмоциях, что не является прямым и достаточно надежным показателем уровня вовлеченности. Эмоции по типу радости или удивления могут быть вызваны сторонними факторами, вместо материала лекции, что ведет к недостоверным метрикам.
* [EduSense](https://www.cmu.edu/news/stories/archives/2019/november/edusense-fitbit-your-teaching-skills.html) : требует множество сенсоров для сбора данных (инфракрасные и видеокамеры, микрофоны), для последующей покадровой/поминутной обработки, вместо анализа длительного поведенческого контекста.

**[ Возможные риски ]**
* Риск избыточного усложнения модели: включение большого количества признаков может не дать прироста в точности, но может создать излишнюю вычислительную нагрузку, что может привести к падению частоты кадров.
* Аппаратные ограничения: локальная обработка может потребовать слишком больших размеров оперативной памяти
* Риск потери обобщающей способности: модель может переобучиться на ограниченном наборе данных, что приведет к резкому падению точности классификации уровня вовлеченности.
